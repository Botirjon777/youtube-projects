# 🚀 What goes viral in coding content (and how we use it)

Notes from researching coding/web Shorts performance in 2026. Source links at the bottom.

## The 3 formats that travel furthest

1. **"You've been doing this wrong."**
   Take a pattern devs accept as normal, show a better way. The emotion = mild embarrassment + relief, and that combo is a *sharing trigger*. → e.g. `Jank Lab` (animating `left` is wrong).

2. **The visible "wow moment."**
   Every Short needs one shareable beat: a metric jumping (2s → 20ms), 20 lines collapsing to 3, or smooth-vs-janky side by side. Build the whole video around that 1 clip.

3. **"The trick that fakes X in N lines."**
   "This CSS trick replaces JavaScript." Short, copyable, screenshot-able.

## What makes the format work

- **Legible code on screen** — zoom in, big font, highlight the line that matters.
- **Split-screen / before–after** — the eye compares instantly. (Both our perf projects do this.)
- **First 2 seconds = the hook.** Show the payoff *before* explaining it.
- **Optimize for SAVES, not watch-time.** Devs pause & rewatch; saves + clicks to long-form are the real signal. End every Short with "Save this."
- **Pair each Short with a long-form** build video → converts viewers to subscribers.

## Hot 2026 topics (high search + shareable)

- Scroll-driven animations, View Transitions API, `:has()`, container queries, subgrid, anchor positioning
- "CSS that replaced JavaScript" micro-interactions
- Performance: jank, reflow/repaint, lazy loading, `content-visibility`
- Explainers: debounce/throttle, event delegation, closures, the event loop
